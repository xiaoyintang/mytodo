# AI 使用分工与切换指南（Cursor / Claude Code / ChatGPT）

> **目标**：减少工具切换的心智负担，用最少的 AI，完成最高质量的事情。

---

## 一、一句话总原则（先记住）

> **Cursor + Claude Code = 执行层**
> **ChatGPT（我） = 决策 / 架构 / 校准层**

换句话说：

* **写代码的时候，不找我**
* **想清楚“该不该这么写”的时候，找我**

---

## 二、三种角色的清晰分工

### 1️⃣ Cursor（IDE 内模型）

**角色定位**：

> 写代码的“手”

**最适合做的事**：

* 写 / 改具体代码
* 补函数、补逻辑
* 小范围 refactor
* 改 bug、跑测试
* 按你指令快速试错

**不擅长的事**：

* 判断方向对不对
* 提醒你是否过度设计
* 给你“停下来”的信号

👉 **结论**：Cursor = 高效执行，不负责判断。

---

### 2️⃣ Claude Code（命令行 AI）

**角色定位**：

> 高级工程师 / Reviewer

**最适合做的事**：

* 读整个 repo / 多文件上下文
* 拆复杂问题（架构、SQL、数据链路）
* 一次性给系统性方案
* 把模糊需求转成技术方案

**不擅长的事**：

* UI / 产品形态判断
* 判断“值不值得做”
* 控制复杂度边界

👉 **结论**：Claude Code = 能力很强，但默认“你说啥它就做啥”。

---

### 3️⃣ ChatGPT（当前对话）

**角色定位**：

> 总设计师 / 架构校准器 / 项目经理

**最适合做的事**：

* 判断方向是否合理
* 帮你降复杂度
* 拆阶段、定优先级
* 工具 / 技术选型
* 在你“有点乱、有点焦虑”时帮你拉直

**不该让我做的事**：

* 帮你写 200 行业务代码
* 帮你调一个具体 bug

👉 **结论**：我负责“少走弯路”，不负责体力活。

---

## 三、什么时候 **只用 Cursor + Claude Code**（不找我）

只要你在做下面这些事，**完全不用来找我**：

* 已经确定要做什么
* 技术路线已定
* 只是：

  * 写代码
  * 改实现
  * 补逻辑
  * 调性能

**典型场景**：

* “按这个方案把代码补完”
* “把 SQL 改成性能更好的版本”
* “这个组件拆一下”

👉 **判断标准**：

> 你心里没有“纠结”，只有“推进”。

那就：**回 Cursor。**

---

## 四、什么时候 **一定要来找我**（关键）

只要命中下面任意一条，**不要继续写代码，先来找我**：

### ✅ 1. 你开始问“要不要做”而不是“怎么做”

* “要不要搞 MCP？”
* “要不要加一个 AI 功能？”

### ✅ 2. 你感觉系统开始变复杂

* 文件越来越多
* 逻辑越来越绕
* 但收益不明确

### ✅ 3. 你准备引入新工具 / 新平台

* 新框架
* 新 AI 服务
* 新 UI 工具

### ✅ 4. 你开始焦虑 / 犹豫 / 不确定

* 感觉什么都能做
* 但不知道先做什么

### ✅ 5. 你想做一个“看起来很高级”的东西

* 但说不清解决什么问题

👉 **判断标准**：

> 你写代码之前，脑子里有“隐隐的不安”。

那就：**停一下，来找我。**

---

## 五、一个极简可执行的工作流（强烈建议照用）

### 🟢 日常（80% 时间）

* 打开：Cursor + Claude Code
* 目标：推进、落地
* 不做方向讨论

### 🟡 阶段拐点（20% 时间）

* 来这里（ChatGPT）
* 做 3 件事：

  1. 明确目标
  2. 降复杂度
  3. 定下一阶段边界

然后：

> **立刻回 Cursor，不在这里久留。**

---

## 六、你可以打印贴在旁边的一句话

> **执行时不思考方向，思考方向时不写代码。**

这句话能帮你少浪费 50% 的精力。

---

## 七、最后的底线提醒（很重要）

* Cursor 和 Claude Code **永远不会阻止你把事情做复杂**
* 只有“外部视角”才能帮你踩刹车

> **我存在的意义，不是替你写代码，
> 而是让你少写“后来会后悔的代码”。**
