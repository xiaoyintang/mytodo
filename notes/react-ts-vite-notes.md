# React + TypeScript + Vite 是什么

## 1) React 是什么
React 是一个用来构建用户界面的前端框架（准确说是 UI 库）。  
它把界面拆成可复用的组件，每个组件负责一小块 UI 和逻辑。

作用：
- 组件化，页面更好拆分和复用
- 状态驱动 UI，数据变了界面自动更新
- 适合做复杂交互的应用

## 2) TypeScript 是什么
TypeScript 是 JavaScript 的“带类型版”。  
它在开发阶段提供类型检查，能提前发现很多错误。

作用：
- 提前发现类型错误，减少运行时崩溃
- 代码更可读，维护成本更低
- IDE 提示更智能，重构更安全

## 3) Vite 是什么
Vite 是前端构建工具（开发服务器 + 打包工具）。  
它负责启动本地开发环境、热更新、打包上线。

作用：
- 启动快、热更新快
- 开发体验更流畅
- 打包更高效

## 为什么比“纯 HTML”更好
纯 HTML + JS 适合简单页面；项目变复杂时会遇到这些问题：
- 逻辑和 UI 混在一起，文件变得难维护
- 复用困难，改一次要到处复制粘贴
- 复杂交互容易出错

React + TypeScript + Vite 的优势：
- 组件化更清晰：功能块可以独立开发和复用
- 类型更安全：TypeScript 能提前发现 bug
- 开发更高效：Vite 启动快，改动即时生效

## 什么时候不一定需要它们
- 只是一个很小的静态页面
- 没有复杂交互和状态管理

## 一句话总结
React 负责搭 UI，TypeScript 负责保证代码可靠，Vite 负责让开发和打包更快。
